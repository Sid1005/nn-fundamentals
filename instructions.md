# How this repo is organized

Each project gets its own folder. Inside that folder is a `README.md` that explains what was built.

## Folder README structure

Follow this order:

1. **What it is.** One sentence. What the project does and what it is built on top of, if anything.
2. **How it works.** The core mechanics, broken into numbered steps. Keep it short. One paragraph per step.
3. **Files.** A table listing each file and what it does.
4. **Live demo.** A link, if there is one.
5. **Run locally.** The command to run it.

## Writing style

- Short sentences.
- No em dashes.
- Plain words. If a simpler word exists, use it.
- Bold the step number and name, then a period. Write the explanation after on the same line.
- File table descriptions are one short phrase, no period at the end.
